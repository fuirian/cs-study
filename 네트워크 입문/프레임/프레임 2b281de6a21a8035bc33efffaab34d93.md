# 프레임

### **프레임**

- LAN에서 데이터를 주고받을 때 사용하는 전송 단위
- 데이터뿐만 아니라 목적지와 출발지의 MAC 주소 정보를 담고 있음

### **이더넷**

- LAN에서 데이터를 전송하기 위한 대표적인 네트워크 기술,
- TCP/IP 모델의 가장 하위 계층인 네트워크 인터페이스 계층에서 동작
- 장치 간에 MAC주소를 기반으로 이더넷 프레임 전송
- **헤더**: 데이터 전송을 위한 부가 정보
- **데이터**: 실제 전송할 데이터
- **트레일러**: 프레임 끝에 붙여 전송 중 데이터가 손상됐는지 확인

### **프리앰블**

- 프레임 전송이 시작되는 것을 알리는 신호
- 출발지와 목적지 컴퓨터는 비트 동기화 진행
    - 비트 동기화: 데이터를 주고 받는 컴퓨터들이 같은 속도로 주고 받을 수 있도록 신호의 클럭을 맞추는 것
- 1010…. 1과 0이 56번 반복

### **SFD**

- 프레임의 시작을 알리는 1바이트의 비트열
- 항상 `1010 1011`
- 바지막 비트가 1 ← 프리앰블의 반복 패턴을 깨고 최로 1이 등장할 때 프레임이 시작한다는 것을 알리는 신호 역할

### **목적지 MAC 주소**

- 프레임을 받을 목적지 컴퓨터의 MAC주소
- 6바이트 구성
- LAN 내 모든 컴퓨터에 브로드캐스트 메시지를 보낼 경우 MAC 주소는 `FF:FF:FF:FF:FF:FF`

### **출발지 MAC 주소**

- 프레임을 보내는 출발지 컴퓨터의 MAC주소
- 6바이트 구성

### **길이/타입**

- 값에 따라 두 가지 다른 의미
- 2바이트로 구성
- 값이 `0~1500`일 때
    - 전송 데이터의 총 길이를 바이트 단위
    - 목적지에서 실제 데이터의 최대 길이를 1500바이트로 제한하기 때문에 전송 데이터의 총 길이
- 값이 `1536` 이상일 때
    - 상위 계층의 프로토콜의 종류를 나타냄

### **데이터**

- 실제 전송 데이터 = 페이로드
- 이더넷 프레임은 전송 데이터의 크기를 최소 46바이트 최대 1500바이트로 제한
- 46바이트보다 작으면 뒤에 0추가하여 46바이트 채움 → 패딩

### **FCS**

- 이더넷 프레임이 오류 없이 전송됐는지 확인하는 오류 검출용 코드
- 총 4바이트
- 목적지와 출발지 MAC 주소, 길이, 데이터 필드의 값을 CRC로 계산한 결괏값
- **CRC**
    - 데이터 전송 중 오류를 검출하기 위해 사용하는 알고리즘
    - 출발지 컴퓨터에서 전송 데이터를 미리 정의된 생성 다항식으로 나눠 나머지를 계산
    - 나머지를 원본 데이터에 붙여 전송
    - 받은 데이터를 같은 생성 다항식으로 나눠 나머지가 0이면 오류가 없다고 판단
    - 0이 아니면 전송 중 오류가 발생했다고 판단
    - 오류 검출만 가능, 정정 불가→재전송 요청
