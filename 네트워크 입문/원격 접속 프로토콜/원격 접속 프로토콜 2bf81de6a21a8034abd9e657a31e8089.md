# 원격 접속 프로토콜

- 물리적으로 멀리 떨어져 있는 컴퓨터에 접속해 시스템을 제어하거나 명령을 실행하기 위한 프로토콜

### 텔넷

- 내 컴퓨터에서 원격 컴퓨터에 접속해 맟 그 컴퓨터 앞에 앉아 작업하는 것처럼 명령을 입력하고 실행 결과를 확인할 수 있게 해주는 프로토콜
- 전송 계층의 TCP 23번 포트 사용
- 멀리 떨어진 서버나 네트워크 장비에 원격으로 접속해 터미널을 통해 필요한 작업 수행 가능
- **터미널**
    - 명령줄 인터페이스 방식으로 명령어를 직접 입력해 사용하는 프로그램
    - 리눅스나 맥OS:  터미널
    - 윈도우: 명령 프롬프트
- 현재 보안상의 이유로 거의 비사용
- 주고받는 데이터가 암호화되지 않은 형태로 전송
- 보안 취약 해결 → SSH 프로토콜
- 네트워크를 통하지 않는 환경에서는 사용

### SSH

- 보안 셸 → 안전한 원격 통신을 하기 위해 사용하는 프로토콜
- 명령줄 인터페이스 방식으로 원격 컴퓨터 시스템 제어
- TCP 22번 포트 사용
- 전 과정을 암호화 채널로 보호하는 원격 접속 프로토콜
- 초기에는 주로 리눅스와 맥OS 터미널 환경에서 원격 시스템에 접속하는데 사용
- 최근 윈도우에서도 OpenSSH 클라이언트 지원
- 공개키 인증 방식의 암호화 기술 사용
- 개인키와 공개키 쌍을 이용해 통신 상대를 안전하게 인증

### RDP

- 클라이언트에서 원격 컴퓨터에 접속해 시스템을 그래픽 사용자 인터페이스 방식으로 제어하는 프로토콜
- 전송 계층 TCP 3,389번 포트 사용
- 마우스를 이용해 제어
- 마이크로소프트가 개발한 독점 프로토콜
- 원격 컴퓨터는 반드시 윈도우 시스템이어야 하지만 클라이언트는 상관없음
- 클라이언트는 마이크로소프트에서 제공하는 원격 데스크톱 연결 프로그램을 실행해 원격 컴퓨터의 IP 주소, 사용자 이름 및 암호를 입력해 접속
- 원격 컴퓨터의 원격 데스크톱 연결 설정 → 허용

### 정리

| **이름** | **설명** | **전송 계층** | **포트 번호** |
| --- | --- | --- | --- |
| 텔넷 | 원격 컴퓨터에 접속해 명령줄 인터페이스 방식으로 시스템을 제어 | TCP | 23번 서버 |
| SSH | 원격 컴퓨터에 암호화된 채널로 접속해 명령줄 인터페이스 방식으로 시스템 제어 | TCP | 22번 서버 |
| RDP | 원격 윈도우 컴퓨터에 접속해 그래픽 사용자 인터페이스 방식으로 시스템 제어 | TCP | 3,389번 서버 |
